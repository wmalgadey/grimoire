# ADR-001: Backend Framework — .NET 9 Minimal API with SignalR

**Status:** Accepted

**Date:** 2026-06

## Context and Problem Statement

The project requires a backend that unifies agent orchestration, channel management (Web UI, Telegram, etc.), and real-time communication. The primary developer has extensive .NET experience. The backend will be developed agentic-first with Spec-Kit — the coding agent generates code based on specifications for developer review.

## Decision Drivers

- Developer productivity through deep framework expertise (critical factor in solo development)
- Real-time requirements: streaming agent responses, channel updates
- SDD compatibility: agent-generated code must be reviewable — domain expertise outweighs marginal LLM performance differences
- Type safety and codebase maintainability

## Considered Options

1. **.NET 9 Minimal API + SignalR** ← chosen
2. Python FastAPI + WebSockets
3. Node.js/TypeScript + tRPC

## Decision Outcome

**Chosen: .NET 9 Minimal API + SignalR**

SignalR natively solves bidirectionality and streaming. Minimal API keeps the codebase lean and highly specifiable for SDD agents (less boilerplate = fewer code generation errors). The LLM performance advantage of Python is not decisive enough for agentic SDD with human review to justify the productivity loss from an unfamiliar framework.

## Consequences

### Positive

- Full control over generated code through framework expertise
- SignalR out-of-the-box for agent streaming and channel sync
- Strong type system reduces spec-to-code drift
- Good testability (xUnit, Moq)

### Negative

- LLM code generation for C# slightly weaker than Python/TypeScript — requires more careful specs
- Smaller open-source ecosystem for AI/LLM libraries (Semantic Kernel as compromise)
